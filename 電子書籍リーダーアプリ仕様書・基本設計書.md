# **電子書籍リーダーアプリ 要求仕様書・基本設計書**

## **1\. システム概要**

ローカルまたは指定領域に保存された電子書籍ファイル（EPUB, CBZ）を読み込み、閲覧するためのAndroidアプリケーション。

進捗管理機能、表示カスタマイズ機能、および縦書き・横書きのスクロール方向制御を備える。

* **アプリ名:** 紡 (Tumugi)  
* **パッケージ名:** cc.namekuji.tumugi

## **2\. 前提技術スタック（推奨）**

* **OS:** Android (Minimum SDK: TBD, Target SDK: TBD)  
* **開発言語:** Kotlin  
* **アーキテクチャ:** MVVM (Model-View-ViewModel) \+ Clean Architecture  
* **UIツールキット:** Jetpack Compose または XML (Composeを推奨)  
* **データベース:** Room (SQLite)  
* **非同期処理:** Kotlin Coroutines / Flow  
* **DIコンテナ:** Hilt または Koin

## **3\. 画面定義・UI要件**

### **3.0 全体UI方針**

* UIはシンプルかつコンパクトな設計とする。  
* 不要なグラデーション等は使用せず、フラットまたはマテリアルデザインに準拠する。  
* アイコンにはモダンなUIライブラリ（Material Symbols等）を採用する。

### **3.1 画面一覧**

1. **本棚画面 (ホーム)**  
2. **設定画面**  
3. **読書画面**

### **3.2 画面詳細仕様**

#### **3.2.1 本棚画面**

* **UI構成:**  
  * トップバー: 「ハンバーガーアイコン」「画面タイトル（本棚）」  
  * タブバー: 「すべて」「EPUB」「CBZ」等のフォーマット別切り替えタブ。  
  * コンテンツ: 書籍およびフォルダのリスト表示（RecyclerView または LazyColumn）  
* **機能要件:**  
  * **フォルダ分け:** ユーザーが任意のフォルダを作成し、書籍を分類できる機能。  
  * **フォーマット別表示:** タブの切り替えにより、EPUBのみ、CBZのみの表示フィルタリングを行う。  
* **表示要素（リストアイテム/カード）:**  
  * 本のタイトル / フォルダ名  
  * 作者名  
  * 読書進捗（% または ページ数/総ページ数）および 読書状態（未読/読書中/読了）  
  * （※ワイヤーにはないが、通常は表紙サムネイルも含む）  
* **アクション:**  
  * ハンバーガーアイコンタップ: ドロワーメニューを開く（「本棚」「設定」等へ遷移）  
  * カードタップ: 対象の書籍の「読書画面」へ遷移、またはフォルダを開く。  
    * 初回起動時: 最初の章（ファイル）の先頭から表示。  
    * 既読時: データベースに保存されたスクロール位置・章から再開（レジューム機能）。

#### **3.2.2 設定画面**

* **UI構成:**  
  * トップバー: 「ハンバーガーアイコン」「画面タイトル（設定）」  
  * コンテンツ: 設定項目のリスト表示  
* **設定項目:**  
  1. **CBZ読書設定**: CBZファイル特有の設定（見開き、画像フィット方法、読み進め方向など）。  
  2. **epub読書設定**:  
     * 表示方向（縦書き/横書きデフォルト）  
     * フォントサイズ、行間、余白設定  
     * フォント変更: ビルトインフォントの選択、およびユーザー端末からの外部フォントファイル（ttf, woff2等）のインポート・適用。  
     * 背景変更: 背景色の指定、およびユーザー端末からの背景画像ファイルのインポート・適用。  
  3. **ページめくり・操作設定**:  
     * 操作許可: スクロールのみ / タップのみ / 両方有効  
     * ページ移動方式: 通常のページ切り替え / 無限スクロール  
     * アニメーション設定: アニメーション再生（ON/OFF）、スクロールアニメーションの種類切り替え。  
     * 音量ボタン操作: 音量ボタンでのページ移動（ON/OFF）。  
  4. **システム・ディスプレイ設定**:  
     * フルスクリーン表示: ナビゲーションバー等の非表示（ON/OFF）。  
     * 画面常時オン: 読書中のスリープ防止（ON/OFF）。  
     * テーマ設定: アプリ全体のテーマ（ライト/ダーク/システム追従）。  
  5. **その他設定**: キャッシュ削除、バージョン情報など。

#### **3.2.3 読書画面**

* **UI構成:**  
  * コンテンツ: 書籍コンテンツ（テキストまたは画像）のフルスクリーン表示（設定に基づく）。  
  * クイックメニュー（オーバーレイ）: 画面中央付近のタップでトグル表示。シンプルでコンパクトなUIとする。  
    * 画面表示などの簡易設定（フォントサイズ、行間、余白の調整、テーマ切り替えなど）を画面遷移せずに直接編集可能。  
    * 戻るボタン（本棚へ）  
* **表示・操作要件:**  
  * **表示方向とスクロール:**  
    * 横書き設定時: 上下スクロール（Vertical Scroll）。  
    * 縦書き設定時: 左右スクロール（Horizontal Scroll）、右から左へ読み進める（RTL）。  
  * **章・ファイル遷移:**  
    * 設定された操作設定（スクロール、タップ、音量ボタン）に基づき、ページまたはファイルの端に達した場合、自動的に次の章またはファイルへ切り替わる。  
  * **状態の保存:**  
    * 読書中のスクロール位置（オフセット値、要素インデックス等）、現在開いている章・ファイルを、スクロール完了時などリアルタイムにデータベースへ保存する。

## **4\. データモデル設計 (Room Entity想定)**

### **4.1 Folder (フォルダテーブル)**

* id (String): フォルダID  
* name (String): フォルダ名  
* parent\_folder\_id (String?): 階層化用（nullはルート）

### **4.2 Book (書籍テーブル)**

書籍のメタデータと進捗を管理する。

* id (String): プライマリキー  
* folder\_id (String?): 所属するフォルダID（nullはルート）  
* title (String): 本のタイトル  
* author (String): 作者名  
* file\_path (String): ローカルファイルへのURI  
* format\_type (Enum): EPUB, CBZ  
* total\_chapters (Int): 総章数または総ページ(画像)数  
* current\_chapter\_index (Int): 現在読んでいる章/画像のインデックス  
* scroll\_position (Int/Float): 現在の章におけるスクロール位置  
* read\_status (Enum): UNREAD, READING, COMPLETED  
* last\_read\_at (Long): 最終閲覧日時  
* finished\_at (Long?): 読了日時

### **4.3 AppSettings (設定テーブル/DataStore)**

* theme\_mode (Enum): Light, Dark, System  
* epub\_font\_size (Int): フォントサイズ  
* epub\_line\_spacing (Float): 行間  
* epub\_margin (Int): 余白  
* epub\_direction (Enum): Vertical(縦書き), Horizontal(横書き)  
* epub\_custom\_font\_uri (String?): ユーザーインポートフォントのURI  
* epub\_background\_color (String): 背景色（Hex等）  
* epub\_background\_image\_uri (String?): ユーザーインポート背景画像のURI  
* cbz\_direction (Enum): LTR, RTL  
* nav\_allowed\_input (Enum): SCROLL\_ONLY, TAP\_ONLY, BOTH  
* nav\_mode (Enum): PAGINATED, CONTINUOUS  
* enable\_animation (Boolean): ページ移動アニメーションの有無  
* animation\_type (Enum): SLIDE, FADE, CURL, NONE 等  
* enable\_volume\_key\_nav (Boolean): 音量ボタンによる移動の有無  
* enable\_fullscreen (Boolean): フルスクリーンモードの有効/無効  
* keep\_screen\_on (Boolean): 画面常時オンの有効/無効

## **5\. 主要な技術的課題と解決方針**

1. **EPUBのパースとレンダリング:**  
   * Folioreader等の既存Android向けEPUBレンダリングライブラリをフォーク・カスタマイズ。  
   * 縦書き表示に問題がある場合はWebView（CSS writing-mode: vertical-rl）へフォールバックする。外部フォント・背景画像の適用はCSSインジェクションで対応する。  
2. **CBZの処理:**  
   * メモリキャッシュ等を有効に活用し（Glide/Coil等）、画像展開・先読みによる動作の高速化を図る。  
3. **状態保存のタイミング:**  
   * 高速なページ移動に対応するため、ライフサイクルだけでなくスクロール完了時等にこまめに非同期で保存処理を実行する。

## **6\. 制約事項・実装してはいけないこと (Constraints & Anti-Patterns)**

* **UIスレッド（メインスレッド）での重い処理の実行**  
  * データベースへの書き込み（頻繁な状態保存など）、EPUBのパース、ZIP（CBZ）の解凍、巨大画像のデコードをUIスレッドで行ってはならない。必ずCoroutines/Flowを用いてバックグラウンドスレッドで処理すること。  
* **全データのオンメモリ化（OOMの誘発）**  
  * CBZ等の画像ファイルを読み込む際、すべてのページを一度にメモリにロードしてはならない。現在表示中のページおよびその前後の先読み領域のみをメモリに保持し、不要になったBitmapは速やかに破棄・再利用すること。  
* **不要なネットワーク通信とバックグラウンド処理の常駐**  
  * 本アプリはローカルファイルの閲覧を主目的とする。ユーザーの許可なくバックグラウンドで不要な通信（トラッキング、テレメトリ等）を行ったり、アプリ画面非表示時にCPUリソースを消費し続けるような常駐サービスを実装してはならない。  
* **仕様に反する複雑なUIや過剰な装飾の追加**  
  * 「シンプルかつコンパクト、無駄なグラデーションを使用しない」という基本方針に反し、意味のない視覚効果や、操作の直感性を損なう複雑なジェスチャー（多本指での特殊スワイプ等）を独自実装してはならない。  
* **プラットフォーム標準のナビゲーションの破壊**  
  * OSの標準的な「戻る」操作（Back Handlerによるシステムナビゲーション）を不当に無効化したり、予期せぬ動作へ書き換えてはならない。  
* **ファイルパスのハードコーディング**  
  * ファイルへのアクセスにおいて、特定の端末やOSバージョンでのみ動作する絶対パスのハードコーディングを行ってはならない。Storage Access Framework (SAF) などを利用した適切なURIベースのファイル処理を実装すること。
